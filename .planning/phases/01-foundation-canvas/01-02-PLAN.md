Create `src/grid/grid.js` exporting a `Grid` class:

**Constructor:**
- `gridSize` property: pixels per foot in world space (default 48 -- gives good visual density)
- `resolution` property: `'1ft'` or `'6in'` (default `'1ft'`)
- `snapEnabled` property: `true` (default)

**`getSpacing()` method:**
- If resolution is `'1ft'`: return `this.gridSize` (one line per foot)
- If resolution is `'6in'`: return `this.gridSize / 2` (two lines per foot, sub-grid at half intervals)

**`draw(ctx, viewport, deltaTime)` method** (matches renderer callback signature):
- Get logical canvas size from `ctx.canvas.width / (window.devicePixelRatio || 1)` for both width and height
- Call `viewport.getVisibleBounds(logicalWidth, logicalHeight)` for culling
- Draw sub-grid (6in marks) when resolution is `'6in'`:
  - Spacing: `this.gridSize / 2`
  - Color: `rgba(255, 255, 255, 0.06)` (subtle on dark background)
  - Line width: `0.5 / viewport.scale` (constant screen-space width)
  - Only draw when zoom scale > 0.3 (hide sub-grid when zoomed far out)
- Draw major grid (1ft marks):
  - Spacing: `this.gridSize`
  - Color: `rgba(255, 255, 255, 0.15)` (visible but not dominant)
  - Line width: `1 / viewport.scale`
- Draw origin crosshair:
  - Color: `rgba(255, 255, 255, 0.3)`
  - Line width: `2 / viewport.scale`
  - Draw horizontal and vertical lines through origin (0,0) spanning visible bounds

**`drawGridLines(ctx, bounds, spacing, color, lineWidth)` private method:**
- Calculate start/end X and Y from bounds, snapped to spacing intervals using floor/ceil
- Use single `beginPath()` for all lines (batched for performance)
- Apply 0.5-pixel offset for crisp rendering: when drawing in world space with viewport transform, use `Math.round(coord) + 0.5 / viewport.scale` offset pattern for sub-pixel alignment
- Draw vertical lines from startX to endX, then horizontal lines from startY to endY
- Single `stroke()` call at end

**`snapToGrid(worldX, worldY)` method:**
- Get current spacing from `getSpacing()`
- Return `{ x: Math.round(worldX / spacing) * spacing, y: Math.round(worldY / spacing) * spacing }`

**`setResolution(resolution)` method:**
- Validates input is `'1ft'` or `'6in'`
- Updates `this.resolution`

Update `src/main.js`:
- Import Grid
- Create Grid instance
- Register `grid.draw.bind(grid)` as a renderer draw callback
- Add grid resolution toggle:
  - Create a minimal UI overlay (positioned fixed, top-right corner)
  - Two buttons or a toggle: "1ft" and "6in"
  - Style: semi-transparent dark background, white text, small font, z-index above canvas
  - On click, call `grid.setResolution('1ft')` or `grid.setResolution('6in')`
  - Highlight active resolution button
- Add snap-to-grid indicator:
  - Track mouse position in world coordinates using mousemove
  - When snap is enabled (default), convert mouse to snapped grid position
  - Register a draw callback that draws a small highlight rectangle at the snapped grid cell (subtle fill, e.g., `rgba(100, 200, 255, 0.15)`, sized to current grid spacing)
  - When Shift key is held (`event.shiftKey` on mousemove), skip snapping -- show indicator at raw world position instead (or hide it)
  - This provides visual feedback that snap-to-grid is working
- Prevent pan from starting when clicking UI overlay buttons (check `event.target` is canvas before starting pan)
