  <name>Task 2: Implement viewport model, coordinate converter, pan/zoom input, and render loop</name>
  <files>

  <action>
Create `src/canvas/viewport.js` exporting a `Viewport` class:
- State: `offsetX = 0`, `offsetY = 0`, `scale = 1`
- `applyTransform(ctx)`: calls `ctx.setTransform(this.scale, 0, 0, this.scale, this.offsetX, this.offsetY)`
- `pan(deltaX, deltaY)`: adds deltas to offsetX/offsetY
- `zoom(zoomFactor, focusX, focusY)`: zoom toward cursor using the standard pattern:
  1. Calculate world position of focus point before zoom: `worldX = (focusX - offsetX) / scale`
  2. Multiply scale by zoomFactor
  3. Recalculate offset: `offsetX = focusX - worldX * scale` (same for Y)
- `getVisibleBounds(canvasWidth, canvasHeight)`: returns `{ topLeft: {x,y}, bottomRight: {x,y} }` in world coordinates
- Constants: `MIN_SCALE = 0.1`, `MAX_SCALE = 10`

Create `src/grid/coordinates.js` exporting a `CoordinateConverter` class:
- Constructor takes a `viewport` reference
- `screenToWorld(screenX, screenY)`: converts using viewport offset and scale
- `worldToScreen(worldX, worldY)`: inverse conversion
- `worldToGrid(worldX, worldY, gridSize)`: returns `{ x: Math.round(worldX / gridSize), y: Math.round(worldY / gridSize) }`
- `gridToWorld(gridX, gridY, gridSize)`: returns `{ x: gridX * gridSize, y: gridY * gridSize }`
- `screenToGrid(screenX, screenY, gridSize)`: composites screenToWorld then worldToGrid

Create `src/canvas/renderer.js` exporting a `Renderer` class:
- Constructor takes `canvasSetup` (CanvasSetup instance), `viewport` (Viewport instance)
- `drawCallbacks` array for registering draw functions (grid will register in Plan 02)
- `start()`: sets `running = true`, calls `requestAnimationFrame`
- `stop()`: sets `running = false`
- `render(timestamp)`:
  1. Calculate deltaTime (ms since last frame)
  2. Get logical size from canvasSetup
  3. Reset transform: `ctx.setTransform(1, 0, 0, 1, 0, 0)`
  4. Scale for DPI: `ctx.scale(dpr, dpr)`
  5. Clear canvas: `ctx.clearRect(0, 0, logicalWidth, logicalHeight)`
  6. Apply viewport transform: `viewport.applyTransform(ctx)`
  7. Call each registered draw callback with `(ctx, viewport, deltaTime)`
  8. If running, request next frame

Create `src/main.js` as the entry point:
- Import CanvasSetup, Viewport, Renderer, CoordinateConverter
- On DOMContentLoaded:
  1. Get canvas element by ID
  2. Create CanvasSetup, Viewport, CoordinateConverter, Renderer instances
  3. Wire pan input: mousedown sets `isPanning = true` and records `lastX/lastY` from `event.clientX/Y`. mousemove (when panning) calculates delta from last position, calls `viewport.pan(deltaX, deltaY)`, updates lastX/lastY. mouseup/mouseleave sets `isPanning = false`. Set `cursor: grab` default, `cursor: grabbing` while panning.
  4. Wire zoom input: wheel event (passive: false), call `event.preventDefault()`. Get canvas-relative coordinates via `getBoundingClientRect()`. Calculate `zoomFactor = 1 - event.deltaY * 0.001`. Clamp: if `viewport.scale * zoomFactor` would exceed MIN_SCALE or MAX_SCALE, return early. Call `viewport.zoom(zoomFactor, canvasX, canvasY)`.
  5. Cache canvas offset for mouse coordinate conversion. Update on resize and scroll events.
  6. Start renderer

Use ES module syntax (`export class`, `import { ... } from`). All imports use relative paths with `.js` extension.
  </action>

  </verify>
