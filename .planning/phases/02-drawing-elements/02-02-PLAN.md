**src/ui/Sidebar.js:**
Create Sidebar class that:
1. Constructor takes `(sidebarElement, coordinateConverter, elementManager, grid)` and stores references
2. Defines element palette data:
   ```
   const PALETTE = [
     { type: 'wall', label: 'Wall', desc: '1ft x 10ft', icon: '|' },

     { type: 'forklift', label: 'Forklift', desc: '2ft x 1ft', icon: 'FK' }

   ```
3. Creates DOM elements for each palette item:
   - Each item is a `div` with padding, border, cursor:grab, margin-bottom
   - Shows icon, label, and description
   - Set `draggable = true`
   - On `dragstart`: `event.dataTransfer.setData('application/json', JSON.stringify({ type }))` and `event.dataTransfer.effectAllowed = 'copy'`
4. Sets up canvas drop handling:
   - `canvas.addEventListener('dragover', e => { e.preventDefault(); e.dataTransfer.dropEffect = 'copy'; })`
   - `canvas.addEventListener('drop', e => { ... })` that:
     a. Reads element type from dataTransfer
     b. Converts drop position to world coords using coordinateConverter.screenToWorld
     c. Snaps to grid using grid.snapToGrid (unless Shift held -- check e.shiftKey)
     d. Imports the correct element class dynamically or uses a factory
     e. Creates element instance at snapped position with default dimensions
     f. Adds to elementManager

**Factory approach (avoid dynamic imports):** Include a static method or module-level function `createElement(type, x, y)` that maps type string to class constructor. Import all shape classes at the top of Sidebar.js.

**IMPORTANT:** The drop event gives clientX/clientY. Convert to canvas-relative coordinates first: `canvasX = event.clientX - canvasRect.left`, then use coordinateConverter.screenToWorld(canvasX, canvasY). Account for the sidebar width offset -- the canvas element's getBoundingClientRect() already handles this since it returns the canvas's actual position.

**src/interaction/Selection.js -- SelectionManager class:**
- Constructor takes `(elementManager)`
- Property: `selectedElement` (null initially)
- `selectElement(element)`: clear previous selection (set .selected = false), set new selection (.selected = true), store reference
- `clearSelection()`: call selectElement(null)
- `deleteSelected()`: if selectedElement exists, call elementManager.remove(selectedElement), set selectedElement to null
- `getSelected()`: return selectedElement
- `drawSelectionOverlay(ctx, viewport, deltaTime)`: if selectedElement exists, draw dashed blue rectangle overlay around it. Use ctx.save()/restore(), strokeStyle = 'rgba(100, 200, 255, 1)', lineWidth = 3 / viewport.scale, setLineDash([6 / viewport.scale, 4 / viewport.scale]), strokeRect with element bounds. Add small resize handles at corners (4x4 filled squares) for visual affordance (no resize behavior yet).

**src/interaction/KeyboardInput.js -- KeyboardController class:**
- Constructor takes `(selectionManager)`
- Sets up window-level keydown listener:
  - Delete or Backspace: `event.preventDefault()`, call `selectionManager.deleteSelected()`. But FIRST check `event.target` -- if target is INPUT, TEXTAREA, or contentEditable, return early (future-proof for property panels).
  - Escape: call `selectionManager.clearSelection()`

**src/main.js modifications:**
1. Import ElementManager, SelectionManager, KeyboardController, Sidebar
2. After existing component initialization, create:
   - `const elementManager = new ElementManager()`
   - `const selectionManager = new SelectionManager(elementManager)`
   - `const keyboardController = new KeyboardController(selectionManager)`
   - `const sidebar = new Sidebar(document.getElementById('sidebar'), coordinateConverter, elementManager, grid)`
3. Pass the canvas element to sidebar so it can set up drop handlers: `sidebar.setupCanvasDrop(canvas)`
4. Register draw callbacks:
   - `renderer.addDrawCallback(elementManager.drawAll.bind(elementManager))` -- draws all elements
   - `renderer.addDrawCallback(selectionManager.drawSelectionOverlay.bind(selectionManager))` -- draws selection overlay on top
   Both AFTER the grid draw callback (already registered) so elements render on top of grid.
5. Add click-to-select on canvas mousedown:
   - In the existing mousedown handler, BEFORE starting pan, do a hit test:
     ```
     const world = coordinateConverter.screenToWorld(canvasX, canvasY);
     const hitElement = elementManager.getElementAtPoint(world.x, world.y);
     if (hitElement) {
       selectionManager.selectElement(hitElement);
       return; // Don't start pan
     } else {
       selectionManager.clearSelection();
       // Continue with pan logic
     }
     ```
   - This means clicking an element selects it (not pan). Clicking empty space clears selection and starts pan.

**CRITICAL: Pan refactor.** The existing mousedown handler in main.js directly sets `isPanning = true`. It needs to be modified to check for element hits FIRST. If an element is hit, select it and do NOT start panning. If no element is hit, clear selection and proceed with panning.

Also pass `canvasRect` to the hit test logic. The mousedown handler already has access to canvasRect (cached in main.js).
