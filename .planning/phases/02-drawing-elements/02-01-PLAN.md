Create src/shapes/ directory and implement the element hierarchy.

**Element.js (base class):**
- Constructor: `(x, y, width, height, type)` -- all in world coordinates (pixels, where 48px = 1 foot)
- Properties: `x, y, width, height, type, selected (false), id (crypto.randomUUID())`
- `containsPoint(worldX, worldY)` -- point-in-rectangle hit test
- `draw(ctx, viewport, deltaTime)` -- base rectangle rendering with ctx.save()/restore(). Fill color varies by selected state. Stroke with scale-invariant line width: `lineWidth / viewport.scale`. Draw type label centered in element using `ctx.fillText()` with scale-invariant font size.
- `getBounds()` -- returns `{x, y, width, height}`
- `toJSON()` -- serializes position, size, type, and type-specific properties (for Phase 4 persistence)

**Wall.js:**
- Extends Element, type = 'wall'
- Default size: 48x480 (1ft x 10ft) -- thin wall segment
- Colors: fill `#555555`, stroke `#333333`
- Draw: solid dark rectangle (wall appearance)

**Office.js:**
- Extends Element, type = 'office'
- Default size: 480x480 (10ft x 10ft)
- Colors: fill `#c4a882` (tan/beige), stroke `#8b7355`
- Draw: filled rectangle with "Office" label

**Rack.js:**
- Extends Element, type = 'rack'
- Default size: 192x48 (4ft x 1ft) -- standard pallet rack depth
- Additional properties: `levels` (default 3), `palletsPerLevel` (default 2)
- Computed: `get totalCapacity()` returns `levels * palletsPerLevel`
- Colors: fill `#4a90d9` (blue), stroke `#2c5f8a`
- Draw: filled rectangle + text showing capacity (e.g., "Rack 3x2=6")
- `toJSON()` includes levels and palletsPerLevel

**Pallet.js:**
- Extends Element, type = 'pallet'
- Default size: 48x40 (48"x40" standard US pallet = 1ft x 0.83ft in grid units, but round to 48x40 world pixels)
- Additional properties: `palletHeight` (default 48, representing 48 inches)
- Colors: fill `#d4a056` (wood/tan), stroke `#9c7430`
- Draw: filled rectangle with "P" label

**Forklift.js:**
- Extends Element, type = 'forklift'
- Default size: 96x48 (2ft x 1ft -- realistic forklift footprint, ~8ft long in real life but simplified)
- Colors: fill `#e8c84a` (yellow), stroke `#b8992e`
- Draw: filled rectangle with "FK" label

**IMPORTANT patterns to follow:**
- All draw methods MUST use ctx.save()/restore() wrapping
- All line widths MUST be scale-invariant: `value / viewport.scale`
- Font sizes MUST be scale-invariant: `${size / viewport.scale}px sans-serif`
- Use world coordinates throughout (48px = 1 foot)
- ES module exports with .js extension imports

  <files>src/managers/ElementManager.js</files>
  <action>
Create src/managers/ directory and implement ElementManager.

**ElementManager.js:**
- Constructor: no arguments, initializes `this.elements = []`
- `add(element)` -- push to array
- `remove(element)` -- splice from array by reference
- `getElementAtPoint(worldX, worldY)` -- iterate in REVERSE order (last element = top z-order) and return first element where `containsPoint()` is true. Return null if none.
- `drawAll(ctx, viewport, deltaTime)` -- iterate FORWARD through elements array, calling `element.draw(ctx, viewport, deltaTime)` for each. This ensures correct z-order (later = on top).
- `getAll()` -- returns elements array (read-only access)
- `clear()` -- empty the array

Export as ES module. No external dependencies.

**Critical detail:** drawAll iterates forward (index 0 first = bottom), getElementAtPoint iterates backward (last index first = top). This ensures the visually topmost element is selected on click.
